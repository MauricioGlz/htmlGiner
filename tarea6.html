Ejercicio 1

Describa los siguientes conceptos de cómputo distribuido
  • RPC

    • En español es llamada de procedimiento remoto, básicamente consiste en la creación de una subrutina o una función que se va a ocupar en mas de una computadora. Se crea como una función cualquiera con la excepción de que esta se ejecutara en una computadora o servidor en específico y puede ser llamada por medio de cualquier computadora que se encuentre en red o que cumpla con los requisitos específicos.

  • CORBA

    • CORBA o "Common Object Request Broker Arquitecture" es un estandar definido para facilitar la comunicación entre diferentes lenguajes y sistemas. A pesar de estar hecho con un enfoque orientado a objetos, ya que se basa en los mismos principios como la encapsulación y la reutilización, los sistemas que trabajan con CORBA rara vez son orientados a objetos.

  • DCOM

    • Distributed Component Object Model, es la tecnología usada por Microsoft para comunicar diferentes componentes en redes distribuidas. DCOM existe para resolver diferentes problemas en los que se encuentran:

    • La estandarización de los objetos que necesitan ser convertidos para ser enviados a través de la red.
    • Asegurarse de que la información del lado del cliente sea desechada al momento de que exista un crash o se pierda la conexión.
    • Especificar una forma de enviar un sin fin de objetos en una sola transmisión para ahorrar ancho de banda.

  • JAVA RMI

    • Java Remote Method Invocation, es una API de JAVA que permite llamadas RPC, es bastante similar a DCOM, con la excepción de que así como DCOM se usa para las diferentes llamadas entre Microsoft este, como su nombre lo indica, se usa entre maquinas virtuales de JAVA.

Ejercicio 2

Realice una tabla comparativa ejemplificando las ventajas y desventajas de cada uno de los conceptos del ejercicio 1
Concepto Ventajas Desventajas
RPC
  • Es independiente del servidor.
  • Permite modelos tanto orientado a procesos como orientado a hilos.
  • Facilita los sistemas distribuidos.
  • Se minimiza el trabajo al momento de la reutilización de código.
  • Permite el uso de aplicaciones distribuidas y no solo en un ámbito local.
  • Permite interoperabilidad entre CORBA y ORB
  • No es un estandar.
  • No resuelve la mayoría de los problemas.
  • Solo se enfoca en la interacción, no ofrece flexibilidad en temrinos de hardware o de arquitectura.
  • Requiere el uso de TCP/IP, cualquier otro medio aun no es soportado
  • Aún no se ha probado en redes de área amplia
CORBA
  • El cliente no necesita saber donde corre una aplicación
  • El cliente no puede encontrar de donde provienen las aplicaciones, para él no existen servidores
  • Las aplicaciones entre cliente y servidor pueden estar escritos en diferentes lenguajes
  • Dificil de hacerlo funcionar en medio de firewalls ya que no permite el mapeo de puertos
  • Las implementaciones varian
DCOM
  • Mecanismo eficiente para el mantenimiento
  • Permite modificar componentes sin afectar la aplicación
  • No es un estandar abierto
  • Es deficiente en un ambiente desconectado
  • Complejo en C++
  • No trabaja bien en medio de firewalls
  • Esta hecho unicamente para Windows
JAVA RMI
  • Facil de empezar
  • Maneja hilos y sockets
  • Carga de clases externas
  • Estrictamente JAVA
  • Problemas de seguridad
  • Problemas cuando la conexión se rompe

Ejercicio 3

Describa el concepto de estilo arquitectónico. Mencione al menos 3 estilos arquitectónico de cómputo distribuido y explica brevemente en qué consiste cada uno de ellos
  • Arquitectura cliente-servidor. Las tarease se reparten en servidores.
  • Arquitectura editor-subscriptor. Parecido al cliente cliente-servidor con la diferencia de que existe un elemento intermedio, en el cliente servidor cada cliente se conecta con el respectivo servidor, en el editor-subscriptor cada cliente esta conectado a un pool de donde se escoge el servidor mas apropiado.
  • Arquitectura P2P. En esta arquitetura no se usan los servidores, en lugar de ello se tienen una serie de computadoras conectadas entre si como iguales
